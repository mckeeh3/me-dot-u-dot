About this App

Me-Dot-U-Dot is an Akka-powered application that explores self-learning AI agents within a two-player board game environment. The core focus is observing how specific agents with their unique models evolve and improve their gameplay strategies over time through experience. Each agent maintains a playbook journal that captures its learning progression, allowing you to track how different models adapt, develop tactics, and refine their decision-making processes.

The application provides valuable insights into model performance characteristics, including response times during each move, which helps compare how quickly various AI models process game situations and make decisions. This makes it an excellent tool for understanding both the learning capabilities and operational efficiency of different AI models in a game-playing context.

The backend uses event-sourced entities, views, and agent components to manage game state, player data, and agent playbooks, while the UI provides a clean, modern experience to create players, start games, and observe the evolution of agent gameplay strategies.

Agent Players

Agent players are AI-powered entities that use large language models to make game decisions. Each agent is uniquely identified by its agent ID and is configured with a specific model (e.g., GPT-4, Claude, etc.).

Agents operate through a structured system prompt that guides their behavior and learning process. They follow a mandatory tool protocol on each turn and maintain a persistent playbook for continuous improvement. Both the playbook and the system prompt are unique to the agent player and are never shared with other agents. Agent Tools

  • Get Game State Tool: Retrieves the current game state as a structured CompactGameState object containing:
    • Game Info: Game ID, creation timestamp, status (empty/in_progress/won_by_player/draw/canceled), and current player ID
    • Players: List of both players with their ID, name, type (human/agent), move count, score, and winner status
    • Board Info: Level (one=5x5 through nine=21x21), top-left square coordinates (A1), and bottom-right square coordinates
    • Move History: Chronological list of all moves with square ID and player ID for each move
  • Get Your Playbook Tool: Accesses the agent's persistent playbook containing self-authored instructions, tactics, and lessons learned from previous games.
  • Get Your System Prompt Tool: Fetches the agent player's current system prompt so the model can review its durable role guidance before deciding how to evolve it.
  • Make Move Tool: Submits a move to the game by specifying board coordinates (e.g., "C3").
  • Update Your Playbook Tool: Allows the agent to rewrite its playbook with new insights, refined strategies, or corrections based on game experience.
  • Update Your System Prompt Tool: Enables an agent player to replace its system prompt with a fully revised version, giving each model the ability to evolve its core role and guardrails over time.
  • Get Game Move History Tool: Returns the enriched move history (including scoring sequences) so agents can perform post-game analysis before revising their playbooks or system prompts.

Learning Process

  • Agents must call tools in a specific sequence: get game state ‚Üí get playbook/system prompt ‚Üí decide move ‚Üí make move ‚Üí optionally update playbook and/or system prompt.
  • The playbook serves as the agent's memory and learning mechanism, storing tactical knowledge that persists across games, while the system prompt captures durable behavioral guidance that the model can refine over time.
  • Agents are instructed to be evidence-driven, balancing exploration of new strategies with exploitation of proven tactics.
  • After each game, agents analyze outcomes (often using the move history tool) and update their playbooks or system prompts with new insights, successful patterns, or failure modes to avoid.

Playbook Viewer

The playbook is the most critical information passed to AI models during gameplay. Along with the current board layout showing which players have selected specific squares, the playbook guides the model's decision-making process for the next move.

The journal entries capture the evolution of an agent's playbook over time. Each entry represents a specific version of the agent's playbook at the moment it was updated. While models always receive the most recent playbook version during gameplay, the journal provides valuable insight into how the agent's strategy and learning evolved across multiple games.

Using the Viewer

  • Select an agent from the dropdown to view their complete playbook journal history.
  • Navigate between journal entries using the up/down arrows to see how the agent's strategy evolved.
  • Each entry displays metadata (Agent ID, Sequence ID, Updated At) and the full playbook instructions from that point in time.
  • Compare entries across time to understand how the agent learned from wins, losses, and gameplay patterns.
  • The viewer uses the same visual style and dropdown components as the game page for consistency.

Agent Role Viewer

The agent role viewer exposes the versioned system prompts that shape each agent's core behavior. While the playbook captures tactical knowledge, the system prompt defines the persistent role and guardrails the agent follows when interacting with the game and tools.

Using the Viewer

  • Select an agent to inspect the historical system prompts attached to that identity.
  • Move between revisions to see how the agent's role definition evolved over time.
  • The diff toggle highlights changes between consecutive prompts to surface meaningful adjustments quickly.
  • Use this history to compare playbook updates with role refinements and understand their combined impact on gameplay.

Game Action Logs

The Game Action Logs page gives you a chronological trace of every move, tool call, and message exchanged during a match. It is ideal for monitoring games that are still in progress or for reviewing the exact sequence of decisions that led to a final result without replaying the board state.

Page Layout

  • Recent Games: Lists the latest matches. Select a row to load its log stream. The refresh button keeps the list up to date while preserving your current selection.
  • Action Stream: Shows paginated log entries for the chosen game with timestamps, player identifiers, action types, and short summaries so you can quickly skim activity.
  • Log Detail: Displays the full text of the selected entry in a scrollable, fixed-width panel, making prompts, tool payloads, or diagnostic messages easy to read.

Usage Tips

  • Click the refresh button to poll for new entries during an active game; the viewer keeps your current page and highlighted log line.
  • Use the up/down pagination buttons to navigate historical entries while the detail pane updates automatically.
  • Combine the action logs with the leader board or replay viewer to correlate narrative events with board positions and outcomes.
